# Plan de Ruta — Funciones faltantes

## Objetivo

Documentar las funciones faltantes del proyecto, priorizarlas, estimar esfuerzo y definir un cronograma claro para su implementación.

## Resumen ejecutivo

Este documento sirve como roadmap para cerrar las funcionalidades pendientes. Contiene una lista priorizada, criterios de aceptación, estimaciones y responsables.

## Cómo usar este archivo

- Actualizar la tabla de "Funciones faltantes" con nuevas entradas.
- Cambiar el campo `Estado` cuando una función avance: `pendiente`, `en progreso`, `bloqueada`, `completada`.
- Añadir enlace a issue/PR en la columna de notas cuando esté disponible.

## Criterios de aceptación

- La función implementada debe tener pruebas unitarias que cubran los casos críticos.
- La interfaz (UI/API) debe cumplir con las especificaciones y pasar revisión de QA.
- Documentación interna añadida en el repo y en este archivo.

## Cronograma y hitos

- Sprint 1 (Semana X): Implementar funciones de prioridad Alta (IDs: ...).
- Sprint 2 (Semana Y): Completar funciones Media y resolver bloqueos.
- Release candidate: Todas las funciones críticas implementadas y testeadas.

## Riesgos y dependencias

- Dependencias externas (paquetes, APIs) pueden retrasar implementaciones.
- Falta de claridad en requisitos puede generar retrabajo — definir criterios de aceptación antes de empezar.
